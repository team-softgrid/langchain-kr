# AI 기반 충전기 AS 사업 통합본

> **문서 버전**: v2.0  
> **최종 수정일**: 2026-04-14  
> **목적**: 전체 사업 방향 정렬 및 내부 공유  

---

## 1. 사업 목표
- 전기차 충전기 AS 운영의 핵심 비효율(오접수, 재출동, 중복출동, 느린 복구)을 데이터 기반으로 줄인다.
- RAG 기반 코파일럿을 중심으로 운영 효율화 + 부가 수익모델을 동시에 구축한다.
- **3년 내 ARR 16억 원, 고객 20곳** 확보를 통해 시장 지배적 포지션 확보

## 2. 시장 기회
- 국내 충전기 **약 50만 기** 설치 (2026.4 기준), 연 7만+ 기 추가
- 충전기 유지보수 시장: **연간 8,000억~1조 원** (2026), 2030년 1.5조 원+
- 정부 정책이 '양 → 질' 전환: O&M 역량이 사업자 선정의 핵심 기준
- 상위 10개 CPO가 시장 72% 점유 → 소수 대형 고객 집중 공략 가능

## 3. 해결하려는 문제
- 24시간 콜센터 운영 부담
- 외주 엔지니어 숙련도 편차
- 오접수 및 불필요 출동
- 부품 미지참/오진에 따른 재방문
- 보증 AS 장기화에 따른 비용 증가

## 4. 통합 솔루션
### 4.1 코어
- AS 이력 기반 RAG 검색/추천
- 원인 Top 1~3, 조치 순서, 필요 부품, 근거 출처 제공

### 4.2 확장 모듈 (순차 투입)
| 순서 | 모듈 | 타이밍 |
|:----:|------|--------|
| 1 | AI 기술콜센터 (24/7 1차 기술응대) | 파일럿 후 3개월 |
| 2 | AS 자재 판매 연계 (권장 부품 자동 제안/주문) | 상용 전환 시 |
| 3 | 충전소 설비 점검관리 SaaS (점검 일정/이력/리포트) | 상용 전환 시 |
| 4 | 지역별 AS 전문가 풀 연결 (권역 매칭/배정) | 상용 안정화 후 |

## 5. 비즈니스 모델
- **★ 핵심**: 월 구독료 (사이트/사용자 기준) — Basic 200~400만, Pro 400~800만, Enterprise 1,000만+
- 구축비: 데이터 정제/인덱스/운영 워크플로우 구축 (1회)
- 사용량 과금: API/관제 연동
- 부가수익: 자재 마진, 전문가 매칭 수수료, 기술콜센터 위탁 운영비
- 성과연동: 절감액 기반 Gain-share (옵션)

## 8. KPI 체계
- 오접수율 / 재출동률 / FTFR / MTTR
- 가동률(Uptime) / 건당 출동 원가
- 1차 해결률(콜센터) / 부품 미지참 재방문율
- 전문가 매칭 소요시간/SLA

## 9. 파일럿 계획 (90일)
- 대상: 사이트 1~2개, 파일럿 비용 약 5,700만 원
- 범위: Top 10 에러코드 + 콜센터/현장 동시 적용
- 확장: 자재 추천 20개 품목, 권역 전문가 풀(최소 5명) 연계
- 성공 기준:
  - 오접수율 15% 이상 감소
  - 재출동률 10% 이상 감소
  - FTFR 8% 이상 상승
  - MTTR 10% 이상 단축

## 10. 자금 계획
| 라운드 | 금액 | 용도 | 마일스톤 |
|--------|:----:|------|---------|
| Seed | 2~3억 | 파일럿 운영 + 팀 구축 | 파일럿 3건 + 유료 1건 |
| Pre-A | 5~8억 | 상용화 + 팀 확대 | ARR 3억+ |
| Series A | 20~30억 | 성장 + 해외 검토 | ARR 10억+ |

## 11. 실행 로드맵 (18개월)
- 1~2개월: 데이터 진단/정제, 정답셋 구축, 파일럿 계약
- 3~5개월: 파일럿 운영 및 KPI 계측
- 6~8개월: 유료 전환 + 레퍼런스 확장
- 9~12개월: 확장 모듈 출시, 팀 확대, ARR 6억
- 13~18개월: Scale-up, 시리즈 A 준비, 해외 검토

## 12. 리스크 및 대응
- 정확도 리스크: Low-confidence 경고 + 사람 승인
- 보안 리스크: Private Cloud/On-Premise 옵션
- 책임 리스크: AI 권고, 최종 판단은 엔지니어 + 법적 면책 조항
- 데이터 품질 리스크: 템플릿 표준화/태깅 규칙 운영
- 규제 리스크: 안전 규정 모니터링 + 법무 자문 체계
- 경쟁 리스크: 데이터 해자 강화 + 빠른 레퍼런스 확보

## 13. 데이터 전략 (Data Moat)
- **플라이휠**: 고객 사용 → AS 이력 축적 → 모델 정확도 향상 → 더 많은 고객 유입
- **데이터 정책**: 고객 데이터 = 해당 고객 전용, 익명화 패턴만 공용 모델 반영
- **IP**: 에러코드 정규화 사전 + 충전기 특화 온톨로지 = 자체 IP
- **목표**: 파일럿 종료 시 10만 건 → 1년 후 50만 건 → 2년 후 200만 건

## 14. 핵심 결론
이 사업은 AI 기능 판매가 아니라 **운영 KPI 개선과 비용 절감 결과를 판매하는 모델**이다.  
연간 8,000억 원 이상의 시장에서 초기엔 코어 RAG + 콜센터/현장 적용에 집중하고,  
데이터 플라이휠을 통해 경쟁 해자를 구축하며 자재/전문가/점검 SaaS로 확장한다.
