# 내부용 문서: AI 기반 충전기 AS 사업 실행안

> **문서 버전**: v2.0  
> **최종 수정일**: 2026-04-14  

---

## 1. 내부 목표
- 90일 내 파일럿 KPI 개선을 증명해 유료 전환
- 6개월 내 레퍼런스 3건 확보
- 12개월 내 ARR 6억 원 달성

## 2. 운영 가설
- RAG 코파일럿이 오접수 및 재출동을 유의미하게 줄인다.
- 콜센터 1차 기술응대 자동화가 야간 인력 부담을 줄인다.
- 출동 전 부품 추천이 미지참 재방문을 줄인다.

## 3. 실행 범위
### 3.1 코어
- 증상/에러코드 기반 추천
- 원인 Top 1~3 + 조치 + 부품 + 근거
- 콜센터/현장 공통 워크플로우

### 3.2 후속 모듈 (순차 투입)
| 순서 | 모듈 | 투입 시기 |
|:----:|------|----------|
| 1 | AI 기술콜센터 | 파일럿 성공 후 3개월 |
| 2 | 자재 판매 연계 | 상용 전환 시 |
| 3 | 점검관리 SaaS | 상용 전환 시 |
| 4 | 전문가 풀 매칭 | 상용 안정화 후 |

## 4. KPI 운영 체계
| 주기 | 지표 |
|------|------|
| 일간 | 질의건수, 낮은 신뢰도 응답 비율 |
| 주간 | 오접수율, 재출동률, FTFR, MTTR |
| 월간 | 절감액, 1차 해결률, 재방문율, ARR |

## 5. 데이터 정책
### 5.1 기본 원칙
- 접수/조치 텍스트 표준화
- 에러코드/부품명 사전 관리
- 피드백(해결/미해결) 필수 수집

### 5.2 데이터 소유권 및 보안
- 고객 데이터 소유권: 해당 고객사에 귀속
- 활용 범위: 계약서에 명시된 범위 내에서만 사용
- 익명화: 개인정보(이름, 연락처, 위치) 제거 후 처리
- 공용 모델: 익명화 패턴 데이터만 반영 (고객 사전 동의 필수)
- 삭제 권한: 계약 해지 시 해당 고객 데이터 완전 삭제

### 5.3 개인정보보호
- 개인정보 처리 방침 수립
- DPO(개인정보보호책임자) 지정
- 접근 로그 기록 및 최소권한 원칙 적용

### 5.4 데이터 확보 목표
| 시기 | 목표 데이터량 | 용도 |
|------|:-----------:|------|
| 파일럿 종료 (3개월) | 10만 건 | 기본 KPI 개선 증명 |
| 상용 1년차 | 50만 건 | 제조사 교차 패턴 분석 |
| 상용 2년차 | 200만 건 | 예측 정비 모델 학습 |

## 6. 조직 운영
- PM: 일정/범위/성과 관리
- 데이터: 정제, 지식베이스 품질관리, 데이터 플라이휠 운영
- 엔지니어링: 검색/응답 품질 개선, 시스템 연동
- 사업개발: 파일럿 계약, ROI 제시, 전환, LoI/MOU 체결

## 10. 상용 전환 기준
- 오접수율 15% 이상 감소
- 재출동률 10% 이상 감소
- FTFR 8% 이상 상승
- MTTR 10% 이상 단축
- **고객사 유료 전환 의사 서면 확인**

## 11. Plan B (파일럿 미달 시)
- **시나리오 A**: KPI 부분 달성 → 코어 RAG 집중, 데이터 재정비 후 2차 파일럿
- **시나리오 B**: KPI 미달 → 점검관리 SaaS를 독립 상품으로 피벗
- **시나리오 C**: 고객 확보 실패 → CSMS 업체에 OEM 기술 라이선싱 전환

## 12. 실행 원칙

4. 데이터 소유권과 보안은 타협 불가
5. 파일럿 성공 전까지 확장 모듈 개발 금지
