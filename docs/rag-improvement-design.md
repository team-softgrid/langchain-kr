# RAG 정확도 개선 설계 문서

> **문서 버전**: v2.0  
> **최종 수정일**: 2026-04-14

## 1. 목적
현재 RAG 프로토타입을 실서비스 수준으로 고도화하기 위해, 아래 5개 개선 축을 설계합니다.
- 에러코드 정규화
- 하이브리드 검색(Dense + Sparse)
- Reranker
- 신뢰도 스코어
- 답변 템플릿 강제

## 2. 권장 운영 전략
### 2.1 기본 권장: 하이브리드 운영
- 검색/인덱스: 로컬
- 생성: 클라우드(OpenAI)

장점:
- 검색 비용 절감
- 민감 데이터 로컬 보관
- 응답 품질 안정성 확보

### 2.2 대안
- 로컬 LLM 100% (Ollama 기반)
- 클라우드 100%

## 3. 개선 항목 상세
### 3.1 에러코드 정규화
목표:
- 서로 다른 표기(`에러코드 23`, `EC23`, `pc7 ec75`)를 동일 키로 통합

설계:
- `normalizer.py` 모듈 추가
- 정규식 룰:
  - `EC\\d+`
  - `에러\\s*코드?\\s*\\d+`
  - `pc\\d+\\s*ec\\d+`
- 동의어 사전:
  - `비상정지 = 긴급정지 = emergency stop`
- 인덱싱 시 메타데이터 필드 추가:
  - `error_code_norm`
  - `symptom_norm`

기대효과:
- 검색 재현율(Recall) 향상
- 동일 장애 패턴 집계 가능

---

### 3.2 하이브리드 검색 (Dense + Sparse)
목표:
- 코드/부품/모델명 같은 exact match 검색 성능 향상

설계:
- Dense: Chroma similarity search
- Sparse: BM25(TF-IDF 포함 가능) 로컬 인덱스
- 스코어 결합:
  - `final = 0.65 * dense + 0.35 * sparse`
- Exact 매칭 보너스:
  - `error_code_norm` 일치 시 +alpha

기대효과:
- 짧은 질의/코드 질의 품질 향상
- 노이즈 문서 비중 감소

---

### 3.3 Reranker
목표:
- 1차 검색 결과 중 상위 문서를 더 정확히 재정렬

설계:
- 1차 후보 Top 30 추출
- Reranker 적용 후 Top 5만 생성 컨텍스트 사용
- 구현 옵션:
  - 클라우드: Cohere/OpenAI 계열 rerank
  - 로컬: BGE reranker 계열

기대효과:
- 생성 컨텍스트 정밀도 상승
- 답변 품질 일관성 향상

---

### 3.4 신뢰도 스코어
목표:
- 답변 신뢰 수준을 계량화해 안전한 응답 제어

설계:
- 신뢰도 계산 요소:
  - 상위 문서 평균 점수
  - 상위 점수 편차
  - 코드 exact match 여부
  - rerank margin(1위-2위 차)
- 임계값 예시:
  - `>= 0.75`: 확신 답변
  - `0.45~0.75`: 제한적 답변 + 확인 권고
  - `< 0.45`: 근거 부족 안내 + 현장 점검 권고

기대효과:
- 환각/과신 응답 감소
- 운영 안전성 강화

---

### 3.5 답변 템플릿 강제
목표:
- 현장에서 바로 쓸 수 있는 고정 포맷 제공

강제 구조:
1. 증상 요약
2. 가능 원인 Top 3
3. 점검 순서
4. 필요 부품
5. 유사 사례 근거
6. 신뢰도/주의사항

설계:
- JSON schema 기반 출력 강제
- UI에서 섹션 렌더링
- 근거 없는 필드 자동 비움 처리

기대효과:
- 응답 일관성 확보
- 감사/추적 용이성 향상

## 4. 아키텍처 제안
- Ingest: 기존 유지 + 정규화 필드 추가
- Index:
  - Dense(Chroma)
  - Sparse(BM25 인덱스)
- Retrieval:
  - Query normalize -> Dense/Sparse 병합 -> Rerank -> Top-N
- Generation:
  - 기본: 클라우드(OpenAI)
  - 옵션: 로컬(Ollama) fallback
- Guardrail:
  - Confidence 기반 응답 레벨 제어

## 5. 로컬 LLM 전환 설계 (Ollama)
### 5.1 최소 변경
- `ChatOpenAI` -> `ChatOllama`
- 임베딩 모델 로컬 대체(예: `nomic-embed-text`)

### 5.2 후보 모델
- `qwen2.5:14b-instruct` (품질/비용 균형)
- 경량 7B 계열 (속도 우선)

### 5.3 고려사항
- GPU/메모리 요구사항
- 응답속도/품질 트레이드오프
- 긴 문맥 처리 성능
